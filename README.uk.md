# Автоматизована система класифікації травм на основі машинного навчання

[![English](https://img.shields.io/badge/lang-en-red.svg)](README.md)

---

Це репозиторій для коду дипломного проєкту на тему **"Автоматизована система класифікації травм на основі машинного навчання"**.

## Опис
Автоматизована система класифікації травм колінного суглоба за допомогою МРТ із використанням методів машинного навчання.

## Вимоги
### Апаратне забезпечення
- **GPU**: підтримка NVIDIA Automatic Mixed Precision (AMP)
- **VRAM**: мінімум 4 ГБ
- **RAM**: протестовано на 16 ГБ
### Програмне забезпечення
- **Python 3.12+** (протестовано на 3.12.1)
- **CUDA драйвер** сумісний із вашим апаратним забезпеченням
- **Основні бібліотеки**:
    - `torch` & `torchvision` (з підтримкою CUDA)
    - `opencv-python` (для попередньої обробки зображень)
    - `numpy`
    - `scikit-learn` (для метрик оцінювання)
    - `pillow`
    - `PyQt6` (для графічного інтерфейсу)

## Головні особливості
- **Двоетапний пайплайн**: Скринінг (бінарна класифікація) -> Діагностика (мультикласова класифікація).
- **Послідовне трансферне навчання (Transfer Learning)**: Тонке налаштування (fine-tuning) від бінарних ваг до спеціалізованого виявлення патологій.
- **Прискорення AMP**: Швидше навчання та зменшення споживання відеопам'яті (VRAM) завдяки змішаній точності.

## Налаштування
Створіть файл `.env` у кореневій директорії проєкту та вкажіть шлях до вашого датасету:

### Змінні середовища

```env
KNEE_CONDITIONS_DATASET=шлях/до/МРТ датасету/з/патологіями
KNEE_DATASET=шлях/до/датасету/з/здоровими/МРТ-зразками

CONVERTED_NPY = data/converted_data/converted_NumPy
CONVERTED_PNG = data/converted_data/converted_PNG
PREPARED_KNEE_DATASET = data/prepared_data
```

### Структура папок
У кореневій папці проєкту (на одному рівні з директорією src/) мають бути створені наступні директорії:

```
.
├── data/           # Сховище сирих та оброблених даних
├── logs/           # Логи роботи програми та навчання
├── models/         # Ваги натренованих моделей (.pth)
└── src/            # Вихідний код
```

### Попередньо натреновані моделі
Для коректної роботи класифікації в графічному інтерфейсі (GUI), у папці models/ повинні бути розміщені наступні файли ваг моделей:
- `knee_3d_binary_model.pth` (Стадія 1)
- `knee_3d_stage2_6classes.pth` (Стадія 2)

## Встановлення

### 1. Налаштування середовища Python
Проєкт базується на версії Python, зазначеній у файлі `.python-version`. При використанні менеджера версій (наприклад, `pyenv`), він автоматично виявить та перемкнеться на необхідну версію. Використовуйте наступну команду для перевірки поточної версії Python:

```Bash
python --version
```

### 2. Створення віртуального середовища
Для ізоляції залежностей проєкту рекомендується використовувати віртуальне середовище (`venv`):

```Bash
# Створення віртуального середовища
python -m venv venv

# Активація (Windows)
venv\Scripts\activate

# Активація (Linux/macOS)
source venv/bin/activate
```

### 3. Встановлення PyTorch з підтримкою CUDA (Важливо!)
Для коректної роботи алгоритмів машинного навчання та використання апаратного прискорення (GPU), необхідно встановити версію PyTorch, яка відповідає вашій версії драйвера CUDA.

Не встановлюйте PyTorch безпосередньо з `requirements.txt`, якщо плануєте використовувати GPU. Замість цього перейдіть на [офіційний сайт PyTorch](https://pytorch.org/get-started/locally/) та оберіть команду для встановлення, що відповідає вашій системі.

Перевірте вашу систему на наявність драйвера CUDA та його конкретну версію. Цілком імовірно, що вам знадобиться встановити CUDA toolkit окремо.

### 4. Встановлення інших залежностей
Після завершення конфігурації PyTorch встановіть решту необхідних пакетів із файлу `requirements.txt`:

```Bash
pip install -r requirements.txt
```
